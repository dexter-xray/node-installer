Удобный Bash-скрипт для быстрой и автоматической установки **Remnawave Node** (демона для панели управления VPN Remnawave) с поддержкой **классического TLS**, автоматическим выпуском SSL-сертификатов и оптимизацией сети.

> **Разработчик:** Dexter | Telegram: [@IamLeonKennedy](https://t.me)

---

## ✨ Возможности скрипта

* ⚙️ **Полная автоматизация:** Скрипт сам обновит пакеты, установит Docker, Docker Compose и Certbot.
* 🔒 **Классический TLS:** Автоматический выпуск и продление SSL-сертификатов от Let's Encrypt (для протоколов вроде *VLESS-Vision-TLS*).
* ⚡ **Оптимизация BBR:** Автоматически включает алгоритм контроля заторов BBR для снижения пинга и увеличения скорости сети.
* 🔄 **Умное автопродление:** Настраивает фоновую задачу (Cron) для проверки сертификатов. Контейнер ноды перезагрузится только тогда, когда сертификат действительно обновится (раз в 2 месяца), не прерывая работу пользователей каждую ночь.
* 📦 **Официальные стандарты:** Полностью соответствует официальной документации Remnawave (использует пути `/opt/remnanode` и структуру контейнера).

## 🚀 Быстрый запуск (Установка в одну строку)

Перед запуском убедитесь, что ваш **домен направлен (A-запись) на IP-адрес сервера**, а порты `80` и выбранный вами порт ноды открыты.

Запустите команду в терминале вашего сервера от имени `root`:

```bash
curl -fsSL https://raw.githubusercontent.com/dexter-xray/node-installer/main/installer.sh | sudo bash
```

## 📝 Пошаговый процесс установки

Во время работы скрипт в интерактивном режиме запросит у вас ключевые данные:
1. **Домен:** Для привязки SSL-сертификата (например, `://example.com`).
2. **Email:** Требуется для регистрации в Let's Encrypt и получения уведомлений о статусе сертификата.
3. **Secret Key:** Секретный ключ, скопированный из вашей панели Remnawave.
4. **Порт:** Порт, на котором нода будет ожидать подключения (По умолчанию: `2222`).

## 🔑 Настройка в панели Remnawave

После успешного завершения работы скрипта, перейдите в вашу веб-панель управления Remnawave. При добавлении ноды или настройке входящего подключения (**Inbound TLS**), укажите следующие локальные пути для Xray-core внутри контейнера:

* 📄 **Путь к сертификату (Certificate Path):** `/var/lib/remnawave/configs/xray/ssl/server.crt`
* 🔑 **Путь к приватному ключу (Key Path):** `/var/lib/remnawave/configs/xray/ssl/server.key`

## 🛠️ Полезные команды для управления

Если вам нужно вручную проверить работу ноды после установки:

* **Просмотр логов ноды:**
  ```bash
  cd /opt/remnanode && docker compose logs -f -t
  ```
* **Перезапуск контейнера:**
  ```bash
  cd /opt/remnanode && docker compose restart
  ```
* **Проверить статус работы BBR:**
  ```bash
  sysctl net.ipv4.tcp_congestion_control
  ```
  *(Должно вернуть: `net.ipv4.tcp_congestion_control = bbr`)*
